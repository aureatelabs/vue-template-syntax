Template Syntax

Vue.js uses an HTML-based template syntax that allows you to declaratively bind the rendered DOM to the underlying Vue instanceâ€™s data. All Vue.js templates are valid HTML that can be parsed by spec-compliant browsers and HTML parsers.

#Text

To pase variable with template you have to use the "Mustache" syntax (double curly braces): 

Message:{{ "{" + "{ textMsg }" + "}" }}

The mustache tag will be replaced with the value of the msg property on the corresponding data object. It will also be updated whenever the data object's msg property changes.

Output:
Message: {{ textMsg }}

#Raw Html

The double mustaches interprets the data as plain text, not HTML. In order to output real HTML, you will need to use the v-html directive: 

{{ "{" + "{ rawHtml }" + "}" }}
{{ "<" }}{{ "span v-html=" }}{{ '"' }}{{ 'rowHtml' }}{{ '"' }}{{ ">" }}{{ "<"}}{{"/"}}{{"span"}}{{ ">" }}

The contents of the span will be replaced with the value of the rawHtml property, interpreted as plain HTML - data bindings are ignored. Note that you cannot use v-html to compose template partials, because Vue is not a string-based templating engine. Instead, components are preferred as the fundamental unit for UI reuse and composition.

Using mustaches: {{ rawHtml }}

#Attributes

Mustaches cannot be used inside HTML attributes. Instead, use a v-bind directive: 

{{ "<" }}{{ "span v-bind:disabled=" }}{{ '"' }}{{ 'isDisabled' }}{{ '"' }}{{ ">" }}{{ "<"}}{{"/"}}{{"span"}}{{ ">" }}
Output:

Disabled Value: {{ isDisabled }}

#Using JavaScript Expressions

So far we've only been binding to simple property keys in our templates. But Vue.js actually supports the full power of JavaScript expressions inside all data bindings: 

{{ "<" }}{{ "p" }}{{ ">" }}{{ "{" + "{ incrementId + 1 }" + "}" }}{{ "<" }}{{ "/" }}{{ "p" }}{{ ">" }}
Output:

Number: {{ incrementId + 1 }}
